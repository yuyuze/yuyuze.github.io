nodejs的事件循环

发表于 2021-10-19 分类于 nodejs ， 基础

小知识：javascript的几乎所有的I/O基元都是非阻塞的。
了解事件循环之前我们要先了解调用堆栈。

调用堆栈

调用堆栈是一个LIFO（后进先出）。 事件循环不断检查调用堆栈，以查看有没有需要运行的函数。 执行时会找到所有的添加到堆栈的函数，并且按照顺序执行每个函数。

ps:

  1. foo进入堆栈的底部
  2. 执行foo的代码，console.log(‘foo’)，运行完移除console.log(‘foo’)
  3. 发现bar（）放入堆栈中，执行bar（）
  4. 目前堆栈中按照先后顺序是foo() bar()
  5. 把console.log(‘bar’)放入堆栈，运行完移除console.log(‘bar’)
  6. 以此类推直到foo的函数结束

    每次迭代中的事件循环都会查看调用堆栈中是否有东西并执行它直到调用堆栈为空

    入队函数执行

    大家都知道，setTimeout是一个宏任务，他是异步的。 按照我的理解是当foo中运行到setTimeout的时候，因为setTimeout是异步的不阻塞代码的运行，定时器时间到了把回调函数放入消息队列中，当foo的代码执行完毕后，执行bar的函数。

消息队列

当调用 setTimeout() 时，浏览器或 Node.js 会启动定时器。 当定时器到期时（在此示例中会立即到期，因为将超时值设为 0），则回调函数会被放入“消息队列”中。

在消息队列中，用户触发的事件（如单击或键盘事件、或获取响应）也会在此排队，然后代码才有机会对其作出反应。 类似 onLoad 这样的 DOM 事件也如此。

事件循环会赋予调用堆栈优先级，它首先处理在调用堆栈中找到的所有东西，一旦其中没有任何东西，便开始处理消息队列中的东西。

我们不必等待诸如 setTimeout、fetch、或其他的函数来完成它们自身的工作，因为它们是由浏览器提供的，并且位于它们自身的线程中。 例如，如果将 setTimeout 的超时设置为 2 秒，但不必等待 2 秒，等待发生在其他地方。

作业队列

Promise使用了该队列（以及基于 promise 构建的 async/await）这种方式会尽快地执行异步函数的结果，而不是放在调用堆栈的末尾。
在当前函数结束之前 resolve 的 Promise 会在当前函数之后被立即执行。

观察者和发布订阅的关系

发表于 2021-10-01 分类于 前端 ， javascript ， 设计模式

设计模式：就是在面向对象开发设计的时候针对一些特定的问题提供简洁优雅的法案。

具有代表性的两个设计模式就是接下来我要提到的观察者模式和发布订阅模式。这两种在我们实际的生活中也是十分常见的。

观察者模式

在这个模式中有两种身份，一种是被观察者（subject）另一种是观察者（observer）。他们之间是一种松耦合的关系。当被观察者的事件改变的时候去执行观察者订阅的方法。下面有张图来说明他们之间的关系。

发布订阅模式

发布者和订阅者是相互不认识的，中间有一个消息队列去处理他们的发布和订阅。是一种完全没有耦合的关系。当订阅者订阅的事件注册到调度中心，发布者发布事件到调度中心触发订阅者的事件。

例如微博这个应用软件，用户觉得该博主的内容很好看去关注他，这个就称为事件的注册，而微博充当了这个调度中心，博主发布了新的文章内容为事件的发布，微博通过轮询注册它的人去把更新消息推送给订阅的人。下面有张图来说明他们之间的关系。

隐式类型转换

字符串和数字之间的类型转化

​ +字符既能用于数字加法,又能用于字符串拼接。当一个操作数是字符串的时候用拼接，当两个操作都是数字的时候用加法。

​ 如果是两个数组相加的话 [1，2，3] + [3 , 4, 5] = “1,2,3,3,4,5”因为一个操作数是对象的时候会执行toPrimitive的抽象操作先看valueOf的返回值能不能得到简单类型，对象不能得到简单类型，再看toString（），[1,2,3]转化为“1，2，3”，再做字符串拼接。

​ -是数字减法运算符。类型会强制转化为数字，* 和 / 两个运算符适用于数字。为了执行减法运算，a和b都需要被转化为数字，他们首先被转化为字符串然后再转化为为数字。

字符串和数字之间的类型转化

​ 0 + false的时候false会隐式强制类型转化为数字。

  4. ？：；中的条件表达式
  5. 逻辑运算符｜｜（逻辑或）和&&（逻辑和） 左边的操作数

他返回的并不是布尔值，而是其中的一个操作数。

  1. ｜｜如果第一个操作数toBoolean返回的是true则返回第一个操作数，如果为false就返回第二个操作数。
  2. &&如果第一个操作数是false就返回第一个操作数，如果为true则返回第二个操作数
符号的强制类型转化

ES6允许从符号到字符串的显示强制类型转化，而隐式类型转化会产生错误。符号不能够被强制转化为数字，但可以被强制转化为布尔值。

宽松相等和严格相等

  • 字符串和数字之间的相等比较

    无论那一端，如果是字符串会强制转化为数字。

  • 其他类型和布尔类型之间的相等比较 tip：布尔值转化为数字进行比较

    （1）如果type（x）是布尔类型，则返回toNumber（x） == y

    （2） 如果type（y）是布尔类型，则返回x == toNumber（y）

  • null和undefined之间的相等比较

    null == undefined //true

  • 对象和非对象之间的相等比较

    如果是字符串和数字跟对象进行比较，x为对象，toprimitive（x） == y

  • 特殊情况

  • 安全运用隐式强制类型转化

    • 如果两边有true或者false的时候尽量不要用==

  • 抽象的比较

网络体系结构

发表于 2021-09-26 分类于 网络规划 ， 网络体系结构

开放式系统互联通信参考模型OSI参考模型(从低到高)

  1. 物理层
  2. 数据链路层
  3. 网络层
  4. 传输层
  5. 会话层
  6. 表示层
  7. 应用层
物理层

为数据链路层实体提供建立、传输、释放所必须的物理连接，并且提供透明的比特流传输。物理层的数据单位是比特，一个二进制位。

  • 物理层的特性

    • 机械特征：规定接口的外形、大小、引脚数和排列固定的位置

    • 电气特征：规定接口电缆上各条线路出现的电压范围

    • 功能特征：指明某条线上出现某一电平的电压表示何种意义

    • 规程特征：指明各种可能事件出现的顺序

    • 数据终端设备（DTE）：提供和接受数据

    • 数据通信设备（DCE）：在DTE和传输线路之间提供信号变化和编码功能

      区别:DCE提供时钟（接口孔头），DTE不提供时钟（接口针头）

数据链路层

将原始的传输线路转变为一条逻辑的传输线路。数据链路层的数据单位是帧，具有流量控制的功能。链路是相邻两结点间的物理线路，数据链路可以理解为数据的通道，是物理链路加上必要的通信协议二组层的逻辑链路。

局域网中的数据链路层可以分为逻辑链路控制和 介质访问控制两个子层。

网络层

控制子网的通信，其主要功能是提供 路由选择，即选择到达目的主机的最优路径，并沿该路径传输数据包。

传输层

传输层利用实现可靠的 端到端的数据传输，实现数据 分段、传输和组装，还提供差错控制和流量/拥塞控制等。

会话层

会话层允许不同机器上的用户之间建立会话。会话就是指各种服务，包括对话控制（记录该由谁来传递数据）、令牌管理（防止多方同时执行同一关键操作）、同步功能

建立和释放会话连接应做的工作：

  • 将会话地址映射到传输层地址
  • 进行数据传输
  • 释放连接
表示层

表示层主要功能有数据语法转化、语法表示、数据加密和解密、数据压缩和解压

应用层

直接针对用户的需要。应用层向应用程序提供服务。

（1）封装

数据在从高层向低层传送的过程中，每层都对接收到的原始数据添加信息，通常附加一个报头和报尾，这个过程叫封装。

（2）网络协议

网络协议由 语法、语义和时序关系三个要素组成。

语法：数据与控制信息的结构或形式

语义：根据需要发出那种控制信息、依据情况完成那种动作以及做出哪种响应。

时序关系：又称同步，即事件实现顺序的详细说明。

什么是类型转换?

将值从一个类型转换为另一种类型通常被称为类型转换，可分为显式类型转换和隐式类型转换。

提到这两个转换，我们首先要提到抽象值操作：字符串、数字和字符串之间类型转换的基本规则。

toString

toString处理非字符串到字符串的强制类型转换。

json.stringify(),json对象序列化为字符串也用到了toString

再对象遇到undefined,function和symbol时自动将其忽略,在数组中则返回null

对包含循环引用的对象使用json.stringify会报错比如o.e = o

如果对象定义了toJson方法,会调用他的返回值序列化.

toNumber:Number()

将非数字转化为数字来使用.true转化为1,false转化为0,undefined转化为NaN,null转换为0

对象(包括数组)会首先被转化为相应的基本基本类型值,如果返回非数字的基本类型,在遵循以上规则进行强制转换

  • 注意: Number(“”) –>0 Number([]) —> 0

toPrimitive

对象会检查该值是否有valueOf()的方法,如果有并且返回基本类型值,就使用该值进行强制类型转化.如果没有就使用toString()的返回值来进行强制类型转化,如果valueof和toSting均不返回基本类型值,会产生TypeError. 使用Object.create(null)创建的对象prototype没有为null,无法进行强制类型转化

toBoolean

javascript中的值可以分为:

  1. 可以强制类型转化为false的值
  2. 其他(被强制类型转化为true)

以下是假值:

  • undefined
  • null
  • false
  • +0,-0和NaN
  • “”
